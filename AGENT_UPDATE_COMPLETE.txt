BREAKING CHANGES FROM v0.1.x
============================

1. HTTP 402 Errors
   - Old: Retried forever
   - New: Stops immediately with ERROR log
   - Action: Check subscription at dashboard

2. Exponential Backoff
   - Old: Immediate retries
   - New: Wait 5s, 10s, 20s, etc between retries
   - Action: Network recovery takes longer but uses less bandwidth

3. HTTP 413 Single Snapshots
   - Old: Retried forever
   - New: Dropped (won't fit in any retry)
   - Action: Reduce snapshot size or increase interval

4. HTTP 207 Parsing
   - Old: Treated as error
   - New: Parsed and logged with details
   - Action: Check logs for rejection details

SUPPORT INFORMATION
===================

Common Issues & Solutions:

Issue: "402 Payment Required" error
  → Cause: Account over host limit or trial expired
  → Solution: Upgrade subscription or remove hosts

Issue: "413 Payload Too Large" for single snapshot
  → Cause: Individual snapshot exceeds 5MB
  → Solution: Reduce metrics collection or increase interval

Issue: "413 Payload Too Large" for batch
  → Cause: Multiple snapshots total > 5MB
  → Solution: Wait for batch to drain or increase interval

Issue: Retries taking longer (5s, 10s, 20s delays)
  → Cause: Exponential backoff on transient errors
  → Note: This is expected and improves reliability

Issue: Snapshots taking time to appear
  → Cause: May be buffered due to network issues
  → Solution: Check network connectivity, agent logs

Issue: Agent not sending any snapshots
  → Cause: Could be API key, endpoint, or billing issue
  → Solution: Check logs, verify configuration, test connectivity
