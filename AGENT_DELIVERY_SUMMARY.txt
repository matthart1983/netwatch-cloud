FEATURES IMPLEMENTED
====================

✅ Feature 1: HTTP 207 Multi-Status Response Handling
   ├─ Parses IngestResponse JSON
   ├─ Extracts accepted/rejected counts
   ├─ Logs individual snapshot rejections
   ├─ Treats partial success as success
   ├─ Handles parse errors gracefully
   └─ Resets failure counter

✅ Feature 2: HTTP 402 Payment Required Handling
   ├─ Detects billing/subscription issues
   ├─ Logs ERROR with actionable message
   ├─ Stops immediately (no retry)
   ├─ Doesn't buffer snapshots
   └─ Clear user guidance in logs

✅ Feature 3: HTTP 413 Payload Too Large Handling
   ├─ Distinguishes batch vs single
   ├─ Batch: Re-buffers for retry
   ├─ Single: Drops immediately
   ├─ Logs with specific details
   └─ Handles both cases safely

✅ Feature 4: Timestamp Validation Documentation
   ├─ Documents ±24 hour window
   ├─ Confirms Utc::now() is safe
   ├─ Provides validation context
   └─ No code changes needed

✅ Feature 5: IngestResponse Parsing
   ├─ Deserializes JSON response
   ├─ Extracts detailed information
   ├─ Logs per-snapshot results
   ├─ Handles errors gracefully
   └─ Info-level success logging

✅ Feature 6: Exponential Backoff Retry Logic
   ├─ Initial delay: 5 seconds
   ├─ Max delay: 300 seconds (5 minutes)
   ├─ Doubles each retry (5, 10, 20, 40, 80, 160, 300...)
   ├─ Detects unrecoverable errors
   ├─ Resets delay on success
   └─ Enhanced logging with timing

BREAKING CHANGES
================

From v0.1.x to v0.2.0:

1. 402 Error Behavior
   ✗ Old: Retried forever
   ✓ New: Stops immediately
   Action: Check subscription

2. Exponential Backoff
   ✗ Old: Immediate retries
   ✓ New: 5s, 10s, 20s... delays
   Action: Network recovery takes longer

3. 413 Single Snapshots
   ✗ Old: Retried forever
   ✓ New: Dropped (won't help)
   Action: Reduce snapshot size

4. 207 Responses
   ✗ Old: Treated as error
   ✓ New: Parsed and logged
   Action: Check logs for details

All changes are documented in code comments.
